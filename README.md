# Хранитель — интеллектуальная система видеонаблюдения для образовательных учреждений

>

## Описание системы

**«Хранитель»** — это интеллектуальная система видеонаблюдения, предназначенная для автоматического обнаружения угроз в образовательных учреждениях. Система анализирует видеопоток в реальном времени, выявляет опасные ситуации и мгновенно оповещает службу безопасности.

В отличие от традиционных систем, которые только записывают видео, «Хранитель» **понимает**, что происходит в кадре: видит оружие, распознаёт драки, слышит крики, замечает падения и возгорания. Это позволяет оперативно реагировать на инциденты, не дожидаясь, пока охранник заметит проблему на одном из десятков мониторов.

---

## Актуальность

Обеспечение безопасности в школах — приоритетная государственная задача. Статистика показывает рост числа инцидентов с насилием и проникновением посторонних. Традиционные системы видеонаблюдения имеют критические недостатки:

- **Человеческий фактор** — охранник физически не может одновременно следить за десятками камер (среднее время концентрации внимания — 20–40 минут).
- **Пассивность** — камеры только записывают, но не анализируют происходящее.
- **Отсутствие масштабируемости** — увеличение числа камер пропорционально растёт нагрузка на персонал.
- **Субъективность** — эффективность зависит от внимательности и опыта конкретного сотрудника.

**«Хранитель»** решает эти проблемы, автоматизируя процесс обнаружения угроз и освобождая человека для принятия решений.

## Возможности

### 🎯 Детекция 6 типов угроз

| Угроза | Описание | Модель |
|--------|----------|--------|
| 🔫 **Оружие** | Обнаружение огнестрельного и холодного оружия в кадре | YOLOv8 (дообучена на 1000 изображений) |
| 👊 **Драка** | Выявление физической потасовки по анализу резких движений | YOLOv8 + анализ последовательности кадров |
| 🏃 **Падение** | Обнаружение человека, внезапно оказавшегося на полу | MediaPipe + LSTM |
| 🔥 **Пожар** | Распознавание огня, дыма и вспышек | YOLOv8 (дообучена на 800 изображений) |
| 📢 **Крик** | Обнаружение криков и звуков паники в аудиопотоке | YAMNet |
| 👥 **Скопление людей** | Выявление аномального скопления с анализом плотности и динамики | YOLOv8 + эвристики |

### 👤 Распознавание лиц
- **Обнаружение лиц**: каскады Хаара (быстрое и эффективное детектирование)
- **Идентификация**: LBPH (устойчив к изменениям освещения, работает на CPU)
- **Точность**: 94.2%
- **Скорость**: 0.12 секунды на одно лицо
- **Применение**: контроль доступа, учёт посещаемости

### 🗺️ Интерактивная карта школы
- Наглядное отображение камер на плане здания
- Переключение между этажами
- Клик по зоне → мгновенный просмотр видео с камеры
- Визуальная обратная связь (анимация при выборе)

### 📊 Оповещения и аналитика
- Единый реестр всех инцидентов
- Фильтрация по статусу (активные / решённые)
- Статистика в виде диаграмм
- Переход к точному моменту инцидента на видео
- Ранжирование угроз по уровню критичности

## Детекция угроз

### 🔫 Оружие
Модель YOLOv8 дообучена на **1000 изображений** (500 с оружием, 500 без). Порог уверенности — 50%. Время до первого оповещения — **0.8 с**.

### 👊 Драка
Основана на анализе резких движений и взаимного расположения людей. Требуется участие минимум двух человек, резкие движения и сближение на расстояние менее 0.5 м в течение 1 секунды.

### 🏃 Падение
Использует MediaPipe для построения скелета (33 точки) и LSTM для анализа изменения позы во времени. Наивысшая точность среди всех детекторов — **93% F1**.

### 🔥 Пожар
YOLOv8 дообучена на **800 изображениях** огня и дыма. Распознаёт как открытое пламя, так и задымление. Порог уверенности — 35% (ниже, чтобы не пропустить раннюю стадию).

### 📢 Крик
Использует готовую модель YAMNet, обученную на 521 классе звуков. Анализирует аудиодорожку видео, выделяя звуки, похожие на крик или панику.

### 👥 Скопление людей
Комбинирует YOLOv8 для подсчёта людей и эвристики:
- Порог количества (настраивается)
- Плотность скопления (среднее расстояние между людьми)
- Динамика изменения (резкий рост количества за 5 секунд)

## Распознавание лиц

Модуль распознавания лиц решает две задачи:
1. **Идентификация** учеников и сотрудников (учёт посещаемости)
2. **Обнаружение посторонних** (контроль доступа)

### Процесс работы:
1. **Добавление персоналий** — администратор вносит данные (ID, ФИО, класс/должность)
2. **Сбор фото** — система делает 100 снимков с веб-камеры
3. **Обучение модели LBPH** — извлекаются текстурные признаки (отпечатки лиц)
4. **Распознавание** — сравнение отпечатка с базой данных, вычисление евклидова расстояния

### Режимы распознавания:
- **По фото** — загрузка одного изображения, мгновенная идентификация
- **По видео** — покадровый анализ с группировкой появлений (интервал < 3 с)

Цветовая индикация:
- 🟢 **Зелёная рамка** — лицо распознано (ученик/сотрудник в базе)
- 🔴 **Красная рамка** — лицо не распознано (посторонний)

## Интерфейс

### 1. Панель управления (Dashboard)
- Карточки камер с живым видео
- Общая статистика (всего видео, обнаруженных угроз, точность)
- Быстрые действия (загрузка видео, аналитика)

### 2. Анализ видео
- Загрузка файла и выбор типов угроз
- Видеоплеер с наложенными рамками
- Список обнаруженных угроз с временными метками
- Переход к нужному моменту по клику

### 3. Оповещения
- Единый реестр всех инцидентов
- Сводка: всего / активных / решённых
- Фильтрация по статусу и поиск по видео
- Кнопки: «Отметить решённым» и «Просмотр»

### 4. Аналитика
- Столбчатая диаграмма частоты угроз по типам
- Данные для управленческих решений

### 5. Карта школы
- Планы этажей с интерактивными зонами
- Клик по зоне → просмотр видео с камеры
- Список камер текущего этажа

### 6. Модуль распознавания лиц
- Персоналии (добавление, редактирование, удаление)
- Фото с камеры (автоматическая съёмка 100 кадров)
- Обучение модели
- Распознавание по фото и видео
- Учёт посещаемости (журнал с фильтром по дате)
